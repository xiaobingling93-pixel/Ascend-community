# 版本出口标准

## 质量要求checklist

| 质量维度     | 首次开源版本                           | RC版本                            | Release版本                                          | 补丁版本                                | 说明                                                                               |
|----------|----------------------------------|---------------------------------|----------------------------------------------------|-------------------------------------|----------------------------------------------------------------------------------|
| 需求E2E可追溯 | 建议提供核心功能需求映射（非强制）                | 关键需求需可追溯至 Issue/PR              | 所有用户可见功能需可追溯                                       | 仅修复问题需关联 Issue                      | 从需求-代码-用例可追溯，所有变更受控并有记录                                                          |
| 社区门禁     | 通过                               | 通过                              | 通过                                                 | 通过                                  | 门禁包含了UT、ST、静态代码检查、编译构建                                                           |
| UT覆盖率    | 语句覆盖率：≥60% ；分支覆盖率：≥50%  （鼓励，不阻塞） | 语句覆盖率：>75%；分支覆盖率：>65%  （牵引，可商榷） | 语句覆盖率：>85% ；分支覆盖率：>75%  （推荐，不强制）                   | 与Release版本一致，不劣化                    | 语句覆盖率和分支覆盖率                                                                      |
| 开源义务履行   | 通过                               | 通过                              | 通过                                                 | 与Release版本一致，不引入新冲突                 | 提供 LICENSE、NOTICE、依赖清单;无 License 冲突（如 GPL 污染）; 提供机器可读 SBOM（SPDX/CDX），第三方组件通过合规审计 |
| 冒烟测试通过率  | 100%                             | 100%                            | 100%                                               | 100%                                |                                                                                  |
| 测试用例通过率  | ≥95%                             | ≥95%                            | ≥98%                                               | 与Release版本一致，不劣化                    |                                                                                  |
| 自动化测试比例  | ≥30% （鼓励，不阻塞）                    | ≥60%（牵引，可商榷）                    | ≥75%（推荐，不强制）                                       | 与Release版本一致，不劣化                    |                                                                                  |
| 安装部署     | 通过                               | 通过                              | 通过                                                 | 与Release版本一致，不劣化                    | 基于社区开源版本部署                                                                       |
| 升级       | NA                               | 通过（支持从最近1个历史Release版本升级）        | 通过（支持从最近2-3个历史Release版本升级，或6-12个月内版本）              | 通过（支持从原Release版本及历史补丁版本升级）          | 满足社区已发布的历史升级                                                                     |
| 性能       | 通过（基础性能验证）                       | 通过（继承特性劣化<5%；新增特性满足基线）          | 通过（继承特性劣化<5%；新增特性满足/优于基线）                          | 与Release版本一致，不劣化                    |                                                                                  |
| 安全       | 通过                               | 通过                              | 通过                                                 | 通过（与Release版本一致，不引入新漏洞）             | 1）通过病毒扫描、漏洞扫描、安全编译选项扫描、开源片段引用扫描、开源软件License合规检查、开源组件扫描；2）通过安全SIG组评审,有正式邮件结论和报告。  |
| 稳定性      | 通过（基础稳定性测试）                      | 通过（仓长稳 + 模型长稳 ≥ 3×24小时）         | 通过（仓长稳 + 模型长稳 ≥ 7×24小时）                            | 通过（回归长稳 ≥ 3×24小时）                   |                                                                                  |
| 兼容性      | 通过（基本硬件/平台兼容）                    | 通过（仓间兼容 + 向前/向后1个minor版本兼容）     | 通过（仓间兼容 + 6-12个月版本兼容）                              | 与Release版本一致，不劣化（仅修复不影响兼容）          |                                                                                  |
| 遗留问题     | 无Blocker/Critical 问题遗留           | 无Blocker/Critical 问题遗留          | 无Blocker/Critical 问题遗留， Low/Medium 问题遗留需通过QA SIG评估 | 无Blocker/Critical问题遗留；仅修复声明问题，不扩大范围 |                                                                                  |
| 资料       | 通过                               | 通过                              | 通过                                                 | 通过（与Release版本一致，更新发布日志）             |

## 版本出口标准

版本发布特性须满足如上质量要求
版本通过兼容性测试/可靠性测试/集成测试
